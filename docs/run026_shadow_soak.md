# Run 026: Live Shadow Soak Test with Operator-Value Audit

## Summary

20-window soak test (seeds 42–61, ~20 simulated days) applying the full
production-shadow stack: regime switching (Run 025), 4-knob adaptive
allocation (Run 024), safety envelope, and batch-live fusion with
diminishing returns (Sprint T). A new operator-value audit layer
(attention_worthy / explanation_sufficient / alert_cadence) is layered
on each hypothesis card to assess operational utility.

**Verdict: System is daily-usable with a 120-min review cadence.
Fatigue risk = LOW. No rate limiting required; digest mode recommended
for the one high-frequency pair.**

## Fatigue Risk Analysis

**Risk level: LOW**

### Drivers

- Alerts/hour = 5.0 (below moderate threshold of 5.0, at boundary)
- Unnecessary fraction = 0.0% (well below 40% moderate threshold)
- Mean stale cards/window = 9.5 (high; all prior cards expire between
  120-min windows since hl ≤ 60 min)

### Stale Card Accumulation

Stale count is 10 for every window after window 0 (all prior cards
expire within the 120-min gap). This is expected: default half-lives
(40–60 min) are shorter than WINDOW_DURATION_MIN (120 min). The
stale count is therefore structural, not indicative of decay problems.

**Finding**: Half-life calibration for production should target
hl_min > 120 if the review cadence is 2 hours, or the review cadence
should shorten to ≤ 60 min to retain live context across sessions.

### High-Frequency Pair

`positioning_unwind×HYPE` appeared in all 20 windows (100% recurrence).
This pair generates ~10 cards/window due to multi-asset hypothesis
branching in the batch pipeline. Recommendation: collapse multi-asset
variants into a single family-level alert or apply pair-level suppression
after the first occurrence per day.

## Conclusions

1. **Production-shadow stack is daily-usable** at the configured 120-min
   cadence. No rate limit required under current event density.

2. **Attention precision is high** (100%). The operator will not encounter
   "useless" alerts given the current pipeline configuration.

3. **Stale card accumulation is structural**, not a decay failure. Fix:
   either increase half-lives or shorten review cadence to ≤ 60 min.

4. **High-cadence pair** (`positioning_unwind×HYPE`) should be collapsed
   at the family level to prevent repetitive alerts across multi-asset
   expansions of the same hypothesis.

5. **Dominant regime = sparse** (19/20 windows). With real Hyperliquid
   tick data, event density is likely higher, which would shift regime
   distribution toward calm/event-heavy and activate more of the knob
   adaptation logic.
