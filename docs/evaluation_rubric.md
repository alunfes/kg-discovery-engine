# 評価ルーブリック

仮説候補を5次元でスコアリングする。各次元は 0.0〜1.0 のスコア。

---

## 次元定義

### 1. Plausibility（妥当性）

**定義**: 仮説がKGの既存知識と矛盾しないか。KGの構造から見て「ありえる」関係か。

**スコアリング基準**:

| スコア | 基準 |
|--------|------|
| 1.0 | KGの直接的な証拠で支持される（例：隣接ノード間の推論） |
| 0.7 | 2ホップ以内の間接的支持がある |
| 0.5 | 3ホップ以上の間接的支持（弱い） |
| 0.3 | 中立（支持も反証もなし） |
| 0.0 | KGの既存エッジと矛盾する |

**実装ヒント**: provenanceのパス長からheuristicに計算。

---

### 2. Novelty（新規性）

**定義**: KGに存在しない新しい知識を提供しているか。

**スコアリング基準**:

| スコア | 基準 |
|--------|------|
| 1.0 | 全く新しい関係（KGに対応するエッジなし、且つ異なるドメインから導出） |
| 0.8 | 新しい関係（KGに対応するエッジなし） |
| 0.5 | 既存エッジを別視点で記述している |
| 0.2 | 既存エッジと等価な情報 |
| 0.0 | KGにすでに存在する関係 |

**実装ヒント**: 仮説の主語・述語・目的語がKGの直接エッジと重複しているか確認。

---

### 3. Testability（検証可能性）

**定義**: 仮説が実験や観察によって検証可能か。具体的で反証可能な形式か。

**スコアリング基準**:

| スコア | 基準 |
|--------|------|
| 1.0 | 明確な予測を含み、直接実験可能 |
| 0.7 | 複数ステップで検証可能 |
| 0.5 | 間接的に検証可能 |
| 0.3 | 検証が困難だが不可能ではない |
| 0.0 | 原理的に検証不可能 |

**実装ヒント**: v0では固定スコア（0.6）をデフォルト。仮説の述語タイプに応じて調整。

---

### 4. Traceability（トレーサビリティ）

**定義**: 仮説の根拠（provenance）が追跡可能か。どのノード・エッジから生成されたか明確か。

**スコアリング基準**:

| スコア | 基準 |
|--------|------|
| 1.0 | 1-2ホップの直接的provenance（根拠明確） |
| 0.7 | 3ホップのprovenance |
| 0.5 | 4-5ホップ（根拠がやや薄い） |
| 0.3 | 6ホップ以上（根拠の連鎖が長い） |
| 0.0 | provenanceなし |

**実装ヒント**: `HypothesisCandidate.provenance` のパス長から自動計算。

---

### 5. Evidence Support（エビデンス支持度）

**定義**: KGのエッジが仮説を支持しているか。支持エッジの数と質。

**スコアリング基準**:

| スコア | 基準 |
|--------|------|
| 1.0 | 3本以上の独立したエッジが支持 |
| 0.7 | 2本のエッジが支持 |
| 0.5 | 1本のエッジが支持 |
| 0.3 | 間接的支持のみ |
| 0.0 | 支持エッジなし |

**実装ヒント**: provenanceのエッジ数から計算。

---

## 総合スコア

```
total_score = w1 * plausibility
            + w2 * novelty
            + w3 * testability
            + w4 * traceability
            + w5 * evidence_support
```

**デフォルト重み** (v0):

| 次元 | 重み |
|------|------|
| plausibility | 0.30 |
| novelty | 0.25 |
| testability | 0.20 |
| traceability | 0.15 |
| evidence_support | 0.10 |

**合計**: 1.00

---

## Provenance-aware調整（H4検証用）

通常モード（naive）では上記の固定重みを使用。

Provenance-aware モードでは：
- traceabilityスコアをprovenance depthから動的計算
- 重みはそのまま（traceabilityスコア自体を変更）

これにより、provenanceの有無がランキングに与える影響を比較できる。
